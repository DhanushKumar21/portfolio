About Me

Building Worlds,
One Blueprint at a Time

I'm a passionate game developer specialising in Unreal Engine 5, currently pursuing my Advanced Diploma in Design & Game Development. I build immersive, mechanic-rich experiences — from atmospheric horror to fast-paced action — and bring the same problem-solving mindset to Unity prototyping and Android app development.

My process usually starts with a single question I want to answer through play — what does it feel like to be hunted, or to solve a puzzle that fights back — and I build the systems, AI, and level design around that question until the answer is something a player can actually feel, not just read about. I care as much about how a Behaviour Tree decision reads in the moment as I do about whether the frame rate holds up under it.

My goal is to join a studio where I can push the limits of interactive storytelling, contribute to high-quality game production, and grow as both a technical and creative developer — while staying hands-on with the kind of systems-level problem solving that got me into games in the first place.

Process

How I Build

Question

Start with the feel

Every project starts with one player feeling I want to create — dread, momentum, satisfaction — before a single system gets built.

Systems

Build the rules, not the script

AI states, lighting rigs, and gameplay loops are built as systems that respond to the player, not scripted one-off moments.

Iterate

Playtest and profile

I test early and often, then profile for performance — a great idea that drops frames isn't shipped, it's reworked.

Polish

Sweat the last 10%

Lighting passes, audio cues, and UI feedback are where a project goes from "working" to "felt" — I budget real time for this.
